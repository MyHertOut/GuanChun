知道《宠魅》这本小说。

它是网络作家“乱”（鱼的天空）创作的异界大陆类玄幻小说，讲述了主角**楚暮**与他的魂宠（如**莫邪**、小蛰龙、白魇魔等）在充满危险与机遇的魂宠世界中，不断战斗、成长、异变，最终走向巅峰的故事。书中关于“魂宠异变”、“魂技搭配”、“属性克制”以及“人宠羁绊”的描写非常精彩，确实非常适合改编成一款深度养成的文字游戏。

既然你要做一款**类似宝可梦的单机文字游戏**，但以《宠魅》为背景，核心设计思路需要体现出原著的独特魅力（如异变、魂力、生存压力），而不仅仅是换皮的宝可梦。

以下是为你构思的**《宠魅：灵魂羁绊》**（暂定名）游戏设计方案：

---

### 一、 核心概念 (Core Concept)

*   **游戏类型**：文字冒险 (Text Adventure) + 策略养成 (RPG) + 开放世界探索
*   **核心体验**：捕捉魂宠 -> 培养/异变 -> 魂宠师进阶 -> 挑战各大势力（魇魔宫、魂殿等）。
*   **与宝可梦的最大区别**：
    1.  **异变系统**：进化不是固定的，低等生物可能因为奇遇异变成高等生物（如月光狐 -> 邪焰六尾妖狐）。
    2.  **魂主参与度**：魂宠师不是旁观者，需要消耗“魂力”释放辅助技能，魂力耗尽会导致魂宠叛逃或战斗失败。
    3.  **生存压力**：魂宠需要吞噬灵核成长，部分魂宠（如魇魔）如果不定期喂养会反噬主人。

---

### 二、 核心系统设计 (System Design)

#### 1. 魂宠系统 (Soul Pet System)
这是游戏的主角。数据结构需要比宝可梦更复杂一些。

*   **种族等级 (Species Rank)**：决定潜力和技能池。
    *   *分级*：奴仆级 -> 战将级 -> 统领级 -> 君主级 -> 帝皇级 -> 主宰级 -> 不朽级。
    *   *设计*：种族等级不是固定的！通过“异变”可以跨越阶级。
*   **成长阶段 (Growth Stage)**：决定当前属性。
    *   *分级*：一阶段 -> 十阶段（每阶段分1-9阶）。
    *   *玩法*：通过战斗获得经验，吞噬“灵核”突破瓶颈。
*   **属性 (Element)**：
    *   风、雷、火、冰、暗、光、妖、兽、虫、木、岩等。
    *   *机制*：双属性或三属性叠加（如莫邪是：兽+火+妖）。
*   **异变 (Mutation)**：
    *   *触发条件*：濒死爆发、吞噬特殊灵物、特定环境、情感羁绊值。
    *   *效果*：种族等级提升，外形改变，属性改变（核心爽点）。

#### 2. 魂宠师系统 (Master System)
玩家扮演的角色。

*   **魂念等级**：决定能契约的魂宠数量和最高等级。
    *   魂徒 -> 魂士 -> 魂师 -> 魂主 -> 魂皇...
*   **魂力 (Soul Power)**：战斗中的MP（魔法值）。
    *   *用途*：召唤魂宠、释放魂技（如“宠魅”技能给魂宠加攻速）、维持魂宠形态。
*   **魂约 (Soul Pact)**：
    *   每个大等级解锁3个魂约位。
    *   *策略*：玩家需要决定携带哪几只魂宠出战（通常1主战，2辅助）。

#### 3. 战斗系统 (Battle System)
采用**回合制文字播报**，但增加战术维度。

*   **战斗流程**：
    1.  **召唤阶段**：消耗魂力召唤魂宠（可同时召唤多只，视魂念而定）。
    2.  **指令阶段**：
        *   魂宠指令：攻击、防御、闪避、释放技能。
        *   魂主指令：移动（拉开距离）、吟唱魂技（加Buff/控制）、喂食药剂。
    3.  **结算阶段**：根据速度和技能优先级生成战斗文本。
*   **文本反馈示例**：
    > *楚暮吟唱咒语，给莫邪施加了“血瞳狂怒”！*
    > *莫邪身上的冕焰暴涨，使用了“冕焰爪”，对敌人造成了暴击！*
    > *敌方魂宠试图反击，但莫邪触发了“月影”特性，闪避了攻击。*

#### 4. 探索与地图 (Exploration)
*   **地图节点**：新月之地、囚岛、万象境、不朽城。
*   **随机事件 (Roguelike元素)**：
    *   *遭遇*：野生魂宠（可捕捉/杀之取核）、流浪魂宠师（PVP）、天材地宝。
    *   *环境*：暴风雨（风系增强）、烈日（火系增强）。

### 三、 数据结构设计示例 (Data Structure)

为了让你更直观地理解如何实现，我写了一个简单的类结构设计（Python风格），包含了注释。

```python
from enum import Enum
from typing import List, Optional

# 1. 定义种族等级
class SpeciesRank(Enum):
    SERVANT = "奴仆级"
    WARRIOR = "战将级"
    COMMANDER = "统领级"
    MONARCH = "君主级"
    EMPEROR = "帝皇级"
    IMMORTAL = "不朽级"

# 2. 定义属性
class ElementType(Enum):
    BEAST = "兽系"
    FIRE = "火系"
    DEMON = "妖灵系"
    ICE = "冰系"
    THUNDER = "雷系"
    # ... 其他属性

# 3. 魂技类
class SoulSkill:
    """
    魂技类，定义技能的基本属性
    """
    def __init__(self, name: str, element: ElementType, power: int, cost: int):
        self.name = name          # 技能名称，如 "冕焰爪"
        self.element = element    # 技能属性
        self.power = power        # 威力
        self.cost = cost          # 消耗体力/妖灵力
        self.description = ""     # 技能描述

# 4. 魂宠类
class SoulPet:
    """
    魂宠基类，包含魂宠的核心数据
    """
    def __init__(self, name: str, species_name: str, rank: SpeciesRank):
        self.name = name                    # 昵称，如 "莫邪"
        self.species_name = species_name    # 种族名，如 "月光狐"
        self.species_rank = rank            # 种族等级
        
        # 基础属性
        self.elements: List[ElementType] = [] # 属性列表，如 [BEAST, FIRE, DEMON]
        self.growth_stage = 1.0             # 成长阶段：1.0 - 10.9
        
        # 战斗属性
        self.health = 100
        self.stamina = 100                  # 体力
        self.loyalty = 80                   # 忠诚度（过低会叛逃/反噬）
        
        self.skills: List[SoulSkill] = []   # 已学会的技能

    def mutate(self, new_species_name: str, new_rank: SpeciesRank):
        """
        异变方法：改变种族和等级，保留部分记忆/技能
        """
        print(f"【系统提示】奇迹发生了！{self.name} 从 {self.species_name} 异变为了 {new_species_name}！")
        self.species_name = new_species_name
        self.species_rank = new_rank
        # 这里可以添加属性修正逻辑

# 5. 魂宠师类
class SoulPetMaster:
    """
    魂宠师类，玩家对象
    """
    def __init__(self, name: str):
        self.name = name
        self.soul_power = 100         # 当前魂力值
        self.max_soul_power = 100     # 最大魂力值
        self.level = "魂士"           # 魂念等级
        self.pets: List[SoulPet] = [] # 契约的魂宠
        self.inventory = {}           # 背包（灵核、魂装）

    def summon(self, pet_index: int):
        """
        召唤魂宠战斗
        """
        if self.soul_power < 10:
            print("魂力不足，无法召唤！")
            return
        
        pet = self.pets[pet_index]
        self.soul_power -= 10
        print(f"{self.name} 念起了咒语，召唤出了 {pet.name}({pet.species_name})！")

# --- 模拟运行 ---
if __name__ == "__main__":
    # 创建主角
    chu_mu = SoulPetMaster("楚暮")
    
    # 获得初始魂宠：月光狐
    mo_xie = SoulPet("莫邪", "月光狐", SpeciesRank.SERVANT)
    mo_xie.elements = [ElementType.BEAST, ElementType.DEMON]
    
    chu_mu.pets.append(mo_xie)
    
    # 模拟战斗召唤
    chu_mu.summon(0)
    
    # 模拟异变事件
    # 假设触发了条件：月光充能 + 濒死状态
    mo_xie.mutate("邪焰六尾妖狐", SpeciesRank.WARRIOR)
```

### 四、 开发建议 (Development Roadmap)

1.  **第一阶段：最小可行性产品 (MVP)**
    *   实现基础的**文字战斗系统**（A打B一下，B扣血）。
    *   实现一只魂宠（月光狐）的**成长**（吃东西升级）和**一段异变**。
    *   只有纯文字界面，靠Log滚动显示剧情。

2.  **第二阶段：内容填充**
    *   建立数据库（JSON/CSV），录入《宠魅》中的经典魂宠（冰空精灵、魔树战士、战也等）。
    *   设计地图系统，允许玩家在“节点”间移动。

3.  **第三阶段：UI与视觉**
    *   虽然是文字游戏，但可以给魂宠配简单的插图或剪影。
    *   添加背景音乐（BGM），战斗时激昂，探索时神秘。
