# ECMAScript (ES) 特性与使用场景指南

本文档列举了 ES6+ 的核心特性及其在现代 Web 开发中的常见使用场景。

## 1. 变量声明 (Let & Const)

- **特性**: 块级作用域变量声明。
- **场景**:
  - `const`: 用于声明不会被重新赋值的常量（如配置项、依赖引用），避免意外修改。
  - `let`: 用于循环计数器或状态变量，替代 `var` 以避免变量提升和作用域污染问题。

## 2. 箭头函数 (Arrow Functions)

- **特性**: 更简洁的函数语法，且不绑定自己的 `this`。
- **场景**:
  - 回调函数（如 `map`, `filter`, `setTimeout`）。
  - 需要保留上下文 `this` 的场景（如 React 组件方法或类方法）。

## 3. 模板字符串 (Template Literals)

- **特性**: 支持多行字符串和内嵌表达式 `${}`。
- **场景**:
  - 构建动态 HTML 片段。
  - 拼接复杂的 API URL 或日志信息。

## 4. 解构赋值 (Destructuring Assignment)

- **特性**: 从数组或对象中提取值赋给变量。
- **场景**:
  - 函数参数解构（如 React Props）。
  - 提取 API 响应中的特定字段。
  - 交换变量值。

## 5. 默认参数 (Default Parameters)

- **特性**: 为函数参数指定默认值。
- **场景**:
  - 初始化配置对象。
  - 防止参数缺失导致的 `undefined` 错误。

## 6. 扩展运算符与剩余参数 (Spread & Rest)

- **特性**: `...` 用于展开数组/对象或收集参数。
- **场景**:
  - 合并数组或对象（浅拷贝）。
  - 函数接收不定数量的参数。
  - 移除对象中的特定属性（配合解构）。

## 7. Promise & Async/Await

- **特性**: 异步编程解决方案。
- **场景**:
  - 处理网络请求（Fetch/Axios）。
  - 文件读写操作。
  - 解决回调地狱（Callback Hell），使异步代码看起来像同步代码。

## 8. 类 (Classes)

- **特性**: 基于原型的面向对象语法糖。
- **场景**:
  - 构建可复用的组件或模块。
  - 定义数据模型（Model）。
  - 实现继承关系。

## 9. 模块化 (Modules - Import/Export)

- **特性**: 标准化的模块加载机制。
- **场景**:
  - 代码拆分与组织。
  - 引入第三方库。
  - 单例模式实现。

## 10. 可选链 (Optional Chaining `?.`)

- **特性**: 安全访问深层嵌套属性。
- **场景**:
  - 访问 API 返回的不确定结构数据（如 `user?.address?.city`）。
  - 避免 `Cannot read property of undefined` 错误。

## 11. 空值合并运算符 (Nullish Coalescing `??`)

- **特性**: 仅在 `null` 或 `undefined` 时提供默认值。
- **场景**:
  - 处理可能为 `0` 或 `false` 的有效值（区别于 `||`）。
  - 配置项默认值回退。

## 12. Map & Set

- **特性**: 新的数据结构。
- **场景**:
  - `Set`: 数组去重。
  - `Map`: 需要非字符串作为键的字典，或频繁增删键值对的场景。

## 13. 数组新方法 (Array Methods)

- **特性**: `includes`, `find`, `flat`, `flatMap` 等。
- **场景**:
  - `includes`: 检查数组是否包含某元素（替代 `indexOf`）。
  - `flat`: 拍平多维数组。
