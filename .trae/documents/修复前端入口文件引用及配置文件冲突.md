修复前端项目启动报错的问题，主要原因是 `index.html` 引用了错误的入口文件 `main.js`（该文件未初始化 Router 和 Pinia），且项目中存在冗余的配置文件。

### 1. 修正入口文件引用
- 修改 `web/index.html`，将 `<script>` 标签的 `src` 属性从 `/src/main.js` 改为 `/src/main.ts`。
  - `main.ts` 中包含了完整的 Vue 实例初始化，包括 `router` 和 `pinia` 的挂载，而 `main.js` 中缺失这些配置。

### 2. 清理冗余和冲突文件
- 删除 `web/src/main.js`：该文件是不完整的入口文件，容易引起混淆。
- 删除 `web/vite.config.js`：该文件缺少路径别名（`@`）和 API 代理配置，应该使用项目现有的 `web/vite.config.ts`，它包含了完整的配置。

### 3. 验证
- 确认修改后前端项目应能正常启动，路由和状态管理可正常工作。
