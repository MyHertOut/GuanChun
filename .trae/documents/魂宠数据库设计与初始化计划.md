# 魂宠数据库设计方案

既然选择从**魂宠数据库**开始，我们将构建整个游戏的核心数据基石。这将不仅仅是一个简单的列表，而是一个包含**种族、等级、属性、技能池、异变路径**的完整关系网。

## 1. 数据库结构设计 (Schema Design)
我们需要定义一个通用的数据结构来描述每一只魂宠。我建议使用 **JSON** 格式，因为它既易于阅读，又方便被 Python 或 Web 前端读取。

### 核心字段定义
*   **基础信息**: ID, 名称, 描述
*   **种族信息**: 种族等级 (奴仆~不朽), 属性 (风/火/雷等)
*   **成长数据**: 基础生命, 基础体力, 成长系数
*   **技能系统**: 自身领悟技能池 (按等级解锁)
*   **异变/进化**: 定义可以进化成什么生物，以及条件
