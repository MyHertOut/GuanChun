# 数据库扩充与系统设计计划

根据你的指示，我们将进行以下四个步骤的开发：

## 1. 建立技能数据库 (`dream/data/skills.json`)
为了将技能逻辑从魂宠数据中解耦，我们将创建一个独立的技能数据库。
*   **结构**: `id`, `name`, `element`, `type` (物理/魔法/变化), `power` (威力), `cost` (消耗), `description`.
*   **内容**: 录入月光狐现有技能，以及即将新增的三只魂宠的核心技能（如：冰墙、龙舞、根须缠绕等）。

## 2. 扩充魂宠数据 (`dream/data/soul_pets.json`)
在现有莫邪（月光狐）的基础上，新增以下三只经典魂宠及其进化形态：
*   **冰空精灵** (ID: 3001): 冰系，战将级。擅长控制与魔法攻击。
*   **小蛰龙** (ID: 2001): 虫/龙/鬼系，统领级。高攻高敏，拥有极强的爆发力。
*   **魔树战士** (ID: 4001): 木系，战将级。高防御与恢复能力，擅长持久战。

## 3. 编写数据校验脚本 (`dream/scripts/verify_data.py`)
创建一个 Python 脚本来验证我们数据的完整性：
*   检查 JSON 格式是否正确。
*   检查 **引用完整性**：确保魂宠配置中的 `skill_id` 在技能数据库中真实存在。
*   输出当前数据库的统计信息（魂宠数量、技能数量）。

## 4. 设计战斗系统 (`dream/3、战斗系统设计.md`)
创建文档，定义核心战斗公式：
*   **伤害公式**: 基于攻击、防御、技能威力、属性克制系数。
*   **速度机制**: 决定出手顺序和闪避概率。
*   **魂力机制**: 魂主如何消耗魂力辅助战斗。
